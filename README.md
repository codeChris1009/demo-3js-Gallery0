## 程式架構與檔案說明

本專案主要檔案功能如下：

- **main.js**
  <br/>
  專案進入點，負責 Three.js 場景、相機、渲染器初始化，建立畫作、邊框、聚光燈、地板反射、左右切換按鈕，並處理動畫循環、互動事件與畫作資訊顯示。

- **arrow-btn.js**
  <br/>
  建立左右切換箭頭按鈕的 3D Mesh，載入透明 PNG 貼圖，並設定顏色空間與自發光效果。

- **click-fn.js**
  <br/>
  處理滑鼠點擊偵測（Raycaster）、判斷點擊目標（左右箭頭），並執行畫廊旋轉動畫（Tween），確保動畫流暢且不重複觸發。

- **texture.js**
  <br/>
  管理畫作資料與貼圖快取，依據索引或 ID 取得對應的圖片與相關資訊，並確保貼圖正確載入與色彩校正。

- **img-border.js**
  <br/>
  建立畫作外框（圓角銀灰色），確保每幅畫作有明顯邊界。

- **spotlight.js**
  <br/>
  建立聚光燈，聚焦於中央畫作，並可調整光強、角度、柔邊等參數。

- **reflector-mirror.js**
  <br/>
  建立地板反射效果，使用 Three.js Reflector 並自訂 shader，模擬拋光黑色地板的真實倒影。

- **helper.js**
  <br/>
  提供數學輔助函式，如計算圓周分布、隨機排序、作品數量等。

- **cursor.js**
  <br/>
  處理自訂滑鼠游標與粒子特效，提升互動體驗。

- **const.js**
  <br/>
  定義畫作資料結構與靜態內容。

本紀錄涵蓋了從零建構具備互動性、平滑動畫與光影特效的 3D 虛擬藝廊（可作為個人作品集展示）的完整技術細節與實作心得。

## 1. 專案建置與 RWD 基礎

- **環境與工具：** 使用 npm 安裝 Three.js，並以 Vite 作為開發與建置工具，利用其即時熱更新的特性提升開發效率。
- **三大渲染核心：** 建立 3D 世界必備的 `Scene`（場景，作為最高層級的容器）、`WebGLRenderer`（渲染器，負責將 3D 轉為 2D 畫面），以及模擬人眼視覺的 `PerspectiveCamera`（透視攝影機）。
- **RWD 視窗適應：** 為了避免視窗縮放造成 3D 物件擠壓變形，必須監聽 `resize` 事件。觸發時除了更新渲染器的尺寸外，**務必同步更新攝影機的長寬比（Aspect Ratio）並呼叫 `updateProjectionMatrix()`**，這是確保畫面不失真的關鍵標準動作。

## 2. 3D 幾何、材質與紋理處理

- **網格架構 (Mesh)：** WebGL 底層運算皆依賴三角形，`Geometry` 負責定義形狀與頂點，`Material` 負責定義外觀，兩者結合成 `Mesh` 才能加入場景。
- **紋理載入與色彩校正：** 透過 `TextureLoader` 讀取外部圖片作為材質紋理（Map）。**重要心得：** 載入的紋理顏色屬性必須明確設定為 `SRGBColorSpace`，否則圖片在 3D 空間中看起來會顯得過於黯淡失真。
- **透明度處理：** 若使用的圖片（如左、右切換箭頭）帶有透明背景，必須在材質設定中明確開啟 `transparent: true`，否則透明部分會顯示為黑色方塊。

## 3. 場景層級架構 (Hierarchy) 與 3D 數學排列

- **父子節點矩陣繼承：** 這是本專案最核心的架構設計。利用沒有實體的 `Object3D` 作為基準節點（Base Node）與根節點（Root Node）。將「畫作」、「黑色背景邊框」與「左右按鈕」設為同一個父節點的子節點，當父節點旋轉或移動時，所有子物件都會繼承其位移與旋轉矩陣（Transform），實現聯動。
- **弧度與環形排列：** 3D 數學運算使用弧度（Radians）而非角度（Degrees）。一個圓為 2π 弧度（約 6.28），為了將 6 幅畫作排成六角形，需利用 `(2 * Math.PI) / 6` 的公式算出每個節點需旋轉的間隔角度。

## 4. 專業光影與反射特效

- **材質升級：** 基礎材質（`MeshBasicMaterial`）不受光照影響，必須改用標準材質（`MeshStandardMaterial`）才能接收光影變化。
- **聚光燈打光 (SpotLight)：** 為了讓視覺聚焦於中央畫作，加入聚光燈並將目標（Target）對準畫作。**核心技巧：** 調整光線邊緣的模糊度 `penumbra` 參數，能產生從中心向左右兩側柔和漸暗的戲劇性打光效果，避免光線邊緣過於生硬。
- **地板倒影 (Reflector)：** 引入 Three.js 內建的 `Reflector` 類別，使用圓形幾何體建立地板。預設反射面是垂直的，必須將其在 X 軸旋轉 90 度（`Math.PI / 2`）使其朝上。**視覺優化心得：** 將反射面的顏色設定為深灰色（如 Hex 碼），可有效降低反射強度，模擬出帶有拋光質感的黑色真實地板，而非刺眼的純鏡面。

## 5. 射線投射互動與 Tween 平滑動畫

- **點擊偵測 (Raycasting)：** 滑鼠點擊的是 2D 螢幕，需藉助 `Raycaster` 射線技術轉換。必須利用公式將滑鼠座標轉換為 -1 到 1 之間的「標準化設備座標 (NDC)」，且因 Y 軸方向相反需加負號。透過遞迴檢查射線與哪些物件相交，並利用自訂的物件 `name` 屬性來判別使用者點擊的是左箭頭還是右箭頭。
- **Tween 動畫過場：** 捨棄生硬的角度跳轉，導入外部 `tween` 模組。設定起始角度、目標角度，並加上 `Quadratic.InOut` 的緩動效果（Easing），創造出滑順的畫廊旋轉體驗。**注意：** 必須在主渲染循環（Animate Loop）中不斷呼叫 `update()`，動畫才會逐格執行。

## 6. UI 疊加與跨域資料管理

- **HTML/CSS 疊加技術：** 不使用耗費效能的 3D 文字，而是直接在 3D Canvas 上層疊加 HTML 的 `<h1>` 與 `<h2>` 標籤（並使用 Google Fonts 增加專業感），配合 CSS 的 `transition: opacity` 來控制淡入淡出。
- **結合 Tween 生命週期：** 利用 Tween 動畫的 `onStart` 鉤子觸發 UI 透明度歸零（淡出），並在 `onComplete` 鉤子觸發透明度恢復（淡入）與更新文字內容。
- **UserData 狀態管理：** 為了在點擊按鈕時立刻知道下一幅畫的資訊，善用了 Three.js Mesh 內建的 `userData` 屬性。在生成左右按鈕時，直接將「上一張 / 下一張」的陣列索引值（Index）預先計算並綁定在按鈕的 `userData` 內，大幅簡化了切換邏輯的複雜度。

## 總結心得

這套架構不僅適用於展示畫作，開發者完全可以將展示圖片替換為「個人專案截圖」，並在點擊中心畫作時加入跳轉至專案詳細頁面的功能，就能打造出一個極具視覺衝擊力與技術深度的 3D 互動式作品集（Portfolio），非常適合作為求職時的亮點展現。
